# Roadmap

The roadmap starts at `0.1.0` and ends at `1.0.0`, the first full compatibility target.

## 0.1.0 — Foundation

- Clean Godot project skeleton.
- Agent instructions and docs.
- Local resource-provider contract.
- Fixed-tick debug sandbox.
- Basic player movement test scene.

## 0.2.0 — Local resources

- Local installation selection.
- Resource index and diagnostics.
- Map, model, texture, sound, and sprite discovery.
- Debug resource browser.

## 0.3.0 — Movement parity

- Fixed simulation player controller.
- Walking, running, jumping, crouch, friction, gravity.
- Numeric debug overlay and replay checks.

## 0.4.0 — Map runtime

- Load local maps into Godot.
- Render and collision path.
- Entity metadata and spawn markers.

## 0.5.0 — First playable item loop

- Inventory and first-person presentation.
- Draw, idle, action, reload state machine.
- Audio and visual event orchestration.

## 0.6.0 — Local round loop

- Teams, spawns, timer, buy phase, live phase, round end.
- Objective scaffolding.
- Scoreboard and basic HUD.

## 0.7.0 — Menus and setup UX

- First-run local installation setup.
- Settings and diagnostics.
- Controller-aware menu navigation.

## 0.8.0 — Bots and director

- Native NPC runtime.
- Perception, memory, navigation abstraction, roles, and director events.
- Debug overlay for decisions.

## 0.9.0 — Cross-platform hardening

- macOS, Windows, Linux export paths.
- Input profiles.
- Resource-path edge cases.
- Release packaging checks.

## 1.0.0 — Compatibility target

- Local setup is usable by normal players.
- Core maps and modes are playable from local resources.
- Movement and timing pass parity checklists.
- Documentation and release notes are complete.
