# Legal Policy

OpenStrike is an independent clean-room engine/client experiment.

The project must not include, copy, distribute, or generate proprietary game code, proprietary assets, trademarks, logos, original UI, branded names, or other protected materials from commercial games.

## Hard rules

- Do not include proprietary source code.
- Do not include leaked or decompiled code.
- Do not include proprietary maps, textures, models, sprites, sounds, fonts, icons, configs, demos, or archives.
- Do not use commercial game trademarks in project identity, UI, branding, or repository assets.
- Do not imitate protected commercial UI, HUD, menus, logos, or visual identity.
- Do not bypass DRM, platform authentication, anti-cheat, or copy protection.
- Do not connect to official servers.
- Do not implement official network protocols.
- Do not load proprietary game logic binaries.
- Do not write into user-provided commercial game directories.

## User-provided local resources

OpenStrike may read compatible resource formats from local files that the user explicitly configures. These directories are mounted read-only.

Users are responsible for ensuring that they have the rights to access the files they configure.

## Repository policy

The repository may contain only:

- original source code;
- original assets;
- assets with explicit open-source licenses;
- synthetic test data;
- documentation;
- tooling.

## README disclaimer

The repository README must keep a clear disclaimer that the project is not affiliated with, endorsed by, sponsored by, or approved by any commercial rights holder.
